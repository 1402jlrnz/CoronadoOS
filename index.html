What is an Operating System?

Definition

An Operating System (OS) is system software that manages computer hardware, software resources, and provides common services for computer programs. It acts as an intermediary between users and computer hardware.

Resource Management

Manages CPU time, memory, storage, and input/output devices

Process Management

Handles creation, scheduling, and termination of processes

Memory Management

Allocates and deallocates memory space to programs

File System

Organizes and manages files and directories

User Interface

Provides interface for users to interact with the computer

Security & Protection

Protects system resources and ensures data security

Examples of Operating Systems

  • Windows: Microsoft Windows 10, Windows 11
  • macOS: Apple's operating system for Mac computers
  • Linux: Ubuntu, Fedora, Debian, Arch Linux
  • Mobile: Android, iOS
  • Server: Windows Server, Linux distributions

Install Windows Server 2008, configure static IP address, ensure proper hardware specifications, and verify network connectivity.

DNS automatically installed with AD → Configure forward lookup zones → Set up reverse lookup zones → Configure root hints.

5. Verify Domain Controller

Check AD DS services, test DNS resolution, verify domain functionality with dcdiag and netdiag commands.

5. RAID Configuration

1. Stripped Volume (RAID 0)

Create stripped volume using Disk Management → Combine multiple physical disks for improved performance. No redundancy - best for temporary data or non-critical applications.

2. Mirrored Volume (RAID 1)

Create mirrored volume for critical system files. Provides exact duplication of data on two disks. Offers complete redundancy with 50% storage efficiency.

3. Spanned Volume

Create spanned volume to combine multiple disks into a single logical volume. Extends storage capacity across multiple drives with no performance improvement.

4. RAID-5 Volume

Create RAID-5 volume with minimum 3 disks. Provides striping with distributed parity for good performance and redundancy. Can survive single disk failure.

5. Test and Monitor RAID

Verify all RAID arrays are functioning properly. Set up monitoring for disk health and RAID status. Test failover scenarios for redundancy verification.

2. Configure Remote Desktop on Workstations

3. Configure Firewall Settings

Open necessary ports (RDP TCP 3389) in Windows Firewall on both server and client workstations.

4. Set Up User Permissions

Add appropriate user groups to "Remote Desktop Users" group on each workstation. Configure per-department access as needed.

5. Test Remote Desktop Connectivity

Test remote connections from server to workstations and vice versa. Verify authentication and proper desktop display.

a. Prohibit access to Control Panel and PC settings
b. Network security: Do not store LAN manager hash value on next password change
c. Prevent access to the command prompt
d. No auto-restart with logged on users for scheduled automatic updates installations
e. All removable storage classes: Deny all accesses
f. Prohibit user install

III. Account Policies - Disable

g. Accounts: Guest account status (Disabled)
h. Minimum password length set to at least 12 characters
i. Maximum password age 42 days

IV. Account Lockout Policy

k. Account lockout duration: 30 minutes
l. Account lockout threshold: 3 invalid logon attempts
m. Reset account lockout counter after: 30 minutes

V. Audit and Access Control

j. Audit directory service access and audit directory service changes
Group Policy Object (GPO) settings should only be accessed by IT admins

VI. Start Menu Restrictions - Users Only

o. Remove Network Connections from the Start Menu: User (Enable), Admin (Disable)
p. Remove Help menu from the Start Menu: User (Enable), Admin (Disable)

r. Remove My Pictures icon from the Start Menu: User (Enable)

t. Remove My Network Places icon from the Start Menu: User (Enable)
u. Add Logoff on the Start Menu: Enable
